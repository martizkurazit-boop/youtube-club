# Лендинг «YouTube — это бизнес» — пакет для публикации на сервере

## Что это
Готовый, работающий лендинг (одностраничный сайт). Это **не макет для переписывания** — это рабочая статическая страница, которую можно сразу выложить на любой сервер/хостинг.

## Структура пакета
```
design_handoff_youtube_landing/
├── index.html          ← главная страница (точка входа)
├── support.js          ← рантайм, который рендерит страницу
├── image-slot.js       ← компонент картинок (drag-and-drop слоты)
├── _ds/                ← дизайн-система (шрифты Gilroy, стили, бандл компонентов)
└── uploads/            ← изображения и GIF, используемые на странице
```

## Как опубликовать (любой из способов)

Это **статический сайт** — серверного кода нет, нужен только хостинг статики.

**Вариант 1 — простой хостинг (Netlify / Vercel / GitHub Pages / Cloudflare Pages)**
1. Загрузите всю папку `design_handoff_youtube_landing/` целиком.
2. Укажите `index.html` как точку входа (обычно определяется автоматически).
3. Готово.

**Вариант 2 — свой сервер (nginx / Apache / любой VPS)**
1. Скопируйте содержимое папки в корень сайта (например `/var/www/html/`).
2. Убедитесь, что `index.html` отдаётся как индексный файл.
3. Откройте домен в браузере.

**Вариант 3 — локальная проверка перед заливкой**
```bash
cd design_handoff_youtube_landing
python3 -m http.server 8000
# открыть http://localhost:8000
```
⚠️ Открывать `index.html` двойным кликом (через `file://`) **не нужно** — часть ресурсов
не загрузится из-за CORS. Всегда отдавайте через HTTP-сервер.

## Важно про пути
Все ссылки внутри `index.html` **относительные** (`./support.js`, `_ds/...`, `uploads/...`).
Поэтому сохраняйте структуру папок как есть. Если кладёте сайт в подпапку
(например `example.com/club/`) — относительные пути продолжат работать.

## Видео
В шапке и в блоке отзывов встроены ролики с YouTube через `<iframe>`:
- Шапка: `https://www.youtube.com/embed/F5BlSf6kPSk`
- Отзыв: `https://www.youtube.com/embed/JA8LpENVb1Y`

Чтобы поменять — найдите в `index.html` строку с нужным `embed/<ID>` и замените ID
на ID нового видео (часть из ссылки `youtu.be/<ID>`).

## ⚠️ Два GIF скачиваются отдельно
Два анимированных GIF (телефоны с нишами и уроками) слишком тяжёлые для архива и
скачиваются отдельными файлами. После распаковки положите их в папку `uploads/`
именно с этими именами:
- `uploads/0626.gif`
- `uploads/IMG_6246.gif`

Без них страница работает, но на двух телефонах в блоке «Что внутри клуба» будут
пустые слоты.

## Картинки
Изображения на странице — это слоты (`<image-slot>`) и обычные `<img>` со значениями
по умолчанию из папки `uploads/`. Чтобы заменить — положите свой файл в `uploads/`
и поправьте `src` в `index.html`, либо перетащите картинку прямо в слот в браузере
(она сохранится в localStorage только для этого браузера, на сервере не сохранится —
для постоянной замены меняйте файлы в `uploads/`).

## Технические детали
- Шрифты: Gilroy (лежат в `_ds/.../assets/fonts/`).
- Внешние зависимости подгружаются с CDN (React и иконки) — нужен доступ в интернет
  у посетителя. Сам сайт хостится статикой.
- Минимальный поддерживаемый размер экрана адаптирован под мобильные (есть медиазапросы).
